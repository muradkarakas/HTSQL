Database Connections

HT/SQL applications need physical database connection(s) in order to execute SQL commands.

The application can have more than one database connection at the same time. For that reason, all connections have a unique name called as "connection name".

New connections are created by create_oracle_connection or create_postgresql_connection function.

"default" connection:

When connection name is not specified, "default" is used as connection name. This happens especially when:

  • A data block does not specify its connection name in Form File (connection-name property).
  • executing SQL and PL/SQL commands in functions.
