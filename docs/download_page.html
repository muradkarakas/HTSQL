v0.5
(Not available)

February 2018

New features:

  • Internet Explorer is now supported.
  • Safari browser is now supported.
  • Opera browser is now supported.
  • Mobile Support: CSS Grid & Flexbox is tested.
  • REDIS support has been added. (12 REDIS functions are now available HT/SQL Built-in Functions).
  • Syntax errors in FRMX file are now reported with line number (Debugging).
  • In order to form a standard method, HT/SQL requires a CSS file for each FRMX file with the same name. For example: HT/SQL adds <link rel="stylesheet" href="logon.css" /> tag into the head section for logon.frmx file.

Bug fix:

  • Thanks to Internet Explorer, many minor bugs have been fixed.
  • create_oracle_connection and create_postgresql_connection functions patched.
  • Implementation of the "setting control block items's value" in the server C code was missing and on the client side, java script had a bug. They patched.

v0.4 February 2018

New features:

  • create_connection function has been replaced with create_oracle_connection and create_postgresql_connection. New functions now return string instead of bool.
  • Demo application is now working with PostgreSQL
  • An installation video has been recorded (Installation Video Guide)
  • HTML5 schema definition file for validation has been added to the the distribution zip file in order to detect syntax error. Developer Tools: HTML5 Schema Definition File for HT/SQL

Bug fix:

  • message function was not working with the string which contains " character. " character in the messageText parameter are escaped.

v0.3 January 2018

New features:

  • run_sql_file function has been added.
  • get_database_name function has been added.
  • get_database_type function has been added.
  • Data block's master-data-block-name property can now be set to Session. If you also set the master-item-name Property of any input tag to a valid session variable name, the input item will be automatically populated with current value of the session variable. This feature is subject to change in future. Please follow the related documentation page.
  • Demo application has been added to the installation package. It only runs on Oracle database for now. But will be ready for PostgreSQL database soon.
  • A new documentation page has been created for Debugging feature of the HT/SQL.

Bug fix:

  • populate_datalist was not work properly for empty datasets for Oracle database. Fixed.
  • The "kill-sesssion" magic button (Magic Buttons) in data block was not redirect the user to the path specified by rediction-path property. Fixed.
  • message function was not working with the string which contains enter character. Enter characters in the messageText parameter are replaced with space now.
  • If the schema object specified by sequence-name Property does not exist, application was crashing. It is now fixed by showing "Sequence does not exist" message.
  • Now html input tags can be closed with /> character sequence without </input> tag.
  • Appearance/functionalities of the buttons in the thead/tfoot tags were problematic. So I have made some corrections in the CSS and java script files.
  • A bug in client side java script related to selection of data block's row has been fixed.
