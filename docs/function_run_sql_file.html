run_sql_file

run_sql_file function has 1 variants:

char run_sql_file(char filePath);

run_sql_file function opens and execute all sql commands in the file sequentially.
Each command must be end wih ';' character.
If error occured during execution, the function return database specific error message and return immediatly.
Database script file path is relative to the htsql.dll file.
Database script file can contain DML or DDL statements but cannot contain begin and end block since Sodium executes each command separated by ';' character.
Example:

void dual.htsql_demo_installation() {
char ret = run_sql_file('htsql_demo_installation.sql');
if (ret is not null) then
message('Error occured : ' || ret);
else
message('Sodium demo schema and objects has been created.');
end if;
}
