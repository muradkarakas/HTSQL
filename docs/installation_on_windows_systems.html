System Resource (Port)

Installation script installs Sodium on port 8090 by default. If this port is used by another application, please find and replace all occurrences of 8090 in Sodium installation script file.

Sodium Installation Steps (<a href="http://www.dataforms.io/htsql_manual/videos/htsql_installation.mp4">Installation Video Guide</a>):

  1. Install database software(s) if not installed. (See Note No.4). If you did any database software installation or made any changes on system environment PATH variable, restart server at this stage.
  2. Download the latest Sodium distribution zip file. (Download Page)
  3. It is advised to create A NEW FOLDER for Sodium installation instead of using the one created before for another reason. (See Note No.5)
  4. Extract Sodium distribution zip file to that folder.
  5. Run the script file HTSQL Install.bat with administrator privilege even though you are logging in as administrator.
  6. At the end of the installation, installation script will open a "Welcome" page. If you see that page, it means ISAPI extension is installed successfully. (See Note No.3) Please bookmark the welcome page.
  7. For database connectivity test, YOU SHOULD RESTART SERVER AT THIS STAGE again. After rebooting, open the bookmarked welcome page and try database connectivity.
Note
  1. On Windows 2016 Server, Oracle Database 11g Express Edition Installer adds the installation path to the system PATH environment variable without ';' character at the end of the folder name.
    Please be sure system environment variable PATH is correct! All paths must be semi-colon separated.
  2. For PostgreSQL database, Tables must be created "with OID" option. Otherwise, data blocks will be read only.
  3. Do not close installation script by clicking "window close icon" of the window. Wait until its processes all commands. It will be closed automatically.
  4. For ORACLE:
    Check the path of the "oci.dll" file is exists in the system environment variable PATH. If not, please add.
    Setting character encoding to UTF8 on the web server is required for Oracle. Otherwise, characters read from database cannot be seen as expected on the client browser. Please set character encoding part of the NLS_LANG value to UTF8 on web server. (Ex: TURKISH_TURKEY.UTF8).
    For POSTGRESQL:
    Check the path of the "libpq.dll" file is exists in the system environment variable PATH. If not, please add.
  5. Do not use space character in installation path.
    Use only ANSI characters in installation path.
  6. Sodium installation path is added to the system environment variable PATH by Sodium installer itself. This is done by a Windows PowerShell script. On some systems, Windows PowerShell may be disabled by default for security reason. If this is the case, add installation path to the system environment variable PATH manually. For example, if your installation path is c:\projects, you must add the path into the path variable as seen in the following picture. (restart is required).
  7. File and folder permission settings.
    Please be sure that Sodium folder, its sub folders and files have been allowed full access to HTSQL user. If not, run Uninstall / Install scripts.
    Installation script waits about 10 seconds for creation of Sodium specific IIS application pool and its security identity account.
  8. Installation script backups IIS configuration settings before modification in case something goes wrong. (https://technet.microsoft.com/en-us/library/dd819405.aspx).
